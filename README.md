
# Программный комплекс сбора цифровых артефактов веб-страниц

Этот проект иллюстрирует архитектуру, описанную в дипломной работе по теме
«Разработка программного комплекса сбора, документирования и анализа цифровых артефактов веб-страниц для систем киберразведки».

## Структура

- `src/`
  - `main.py` — точка входа, парсинг аргументов, запуск обработки.
  - `config.py` — описание конфигурации (`Config`) и её формирование.
  - `logging_setup.py` — настройка логирования.
  - `browser.py` — создание и завершение Selenium WebDriver.
  - `visual_artifacts.py` — сохранение скриншота и HTML.
  - `network_artifacts.py` — извлечение performance-логов (сетевые артефакты).
  - `ssl_info.py` — получение и сохранение информации о TLS-сертификате.
  - `integrity.py` — функции хеширования (SHA256).
  - `storage.py` — чтение списка URL, формирование директорий и имён файлов.
  - `executor.py` — параллельная обработка списка URL.
- `data/`
  - `sample.txt` — пример файла со списком URL.
- `results/`
  - сюда записываются артефакты конкретных запусков.
- `requirements.txt` — зависимости Python.

## Установка и запуск

1. Создать и активировать виртуальное окружение (по желанию).
2. Установить зависимости:

```bash
pip install -r requirements.txt
```

3. Заполнить файл `data/sample.txt` URL-адресами (по одному в строку).
4. Запустить обработку:

```bash
python -m src.main --input data/sample.txt --threads 4
```

Результаты будут сохранены в директории `results/run_YYYY-MM-DD_HH-MM-SS/`.
